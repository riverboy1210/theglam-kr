## 2. 기술 아키텍처 (Technical Architecture)

### 2.1 프론트엔드 스택
- **Framework**: Next.js 14+ (App Router)
- **Rendering Strategy**: 
  - 정적 콘텐츠: SSG (Static Site Generation)
  - 빈번한 업데이트 콘텐츠: ISR (Incremental Static Regeneration)
  - 사용자 맞춤형 데이터: Client-side Rendering (React Query)
- **Styling**: Tailwind CSS / Framer Motion (애니메이션)

### 2.2 SEO 및 성능 최적화
- **Metadata**: `generateMetadata` API를 통한 동적 메타 태그 생성
- **Sitemap**: `sitemap.ts`를 통한 자동 생성 및 Google Search Console 연동
- **Structured Data**: JSON-LD (Article, Hospital, Review, FAQ 스키마 적용)
- **Core Web Vitals 목표**:
  - LCP (Largest Contentful Paint): ≤ 2.5s
  - INP (Interaction to Next Paint): ≤ 200ms
  - CLS (Cumulative Layout Shift): ≤ 0.1

### 2.3 인프라 및 도메인
- **Hosting**: Vercel 또는 AWS Amplify
- **Domain**: theglam.kr (기존 Webflow .io 도메인에서 완전 전환 및 리다이렉트 설정)

## 3. 콘텐츠 마이그레이션 전략 (Content Migration Plan)

### 3.1 콘텐츠 분류 및 재구성
기존 383개의 블로그 글을 4가지 핵심 유형으로 재분류하여 구조화합니다.
- **PRICE (비용비교 - 40개 목표)**: 시술별 평균 비용 및 지역별 가격 편차 분석
- **SAFETY (안전정보 - 40개 목표)**: 부작용 방지 가이드, 의료 사고 예방 수칙
- **RECOVERY (회복관리 - 126개 목표)**: 시술 후 관리법, 붓기 제거, 일상 복귀 팁
- **CLINIC_CHOICE (병원선택 - 177개 목표)**: 지역별 병원 특징 및 전문의 확인법

### 3.2 변환 파이프라인 (Transformation Pipeline)
- **데이터 추출**: 기존 Webflow/Blog 데이터 JSON 변환
- **구조화**: `TRANSFORMATION_PIPELINE.md` 규칙에 따른 마크다운 변환
- **이미지 최적화**: WebP 포맷 변환 및 Next.js Image 컴포넌트 적용
- **링크 검증**: 내부 링크 구조 재설정 및 깨진 링크 수정

### 3.3 URL 리다이렉트 전략
- **기존**: `theglam.webflow.io/post/old-slug`
- **신규**: `theglam.kr/articles/new-slug`
- **방법**: Next.js `next.config.js` 내 `redirects` 설정을 통한 301 영구 리다이렉트 구현

## 4. 웹사이트 10대 섹션 상세 기획 (10-Section Plan)

### [Section 1] 홈 (Index / Home)
- **섹션 목적 & 사용자 여정**: 브랜드 첫인상 구축 및 핵심 서비스로의 빠른 이동 유도. 사용자가 현재 가장 인기 있는 시술 정보나 병원 정보를 한눈에 파악하도록 함.
- **핵심 기능 리스트**:
  - 메인 비주얼 배너 (프로모션 및 브랜드 메시지)
  - 실시간 인기 검색어 및 시술 카테고리 퀵 메뉴
  - 추천 콘텐츠 카드 (에디터 픽)
  - 최근 등록된 리얼 후기 프리뷰
- **콘텐츠 전략**: 주간 단위 트렌드 반영, 고해상도 브랜드 이미지 사용.
- **SEO 전략**: 메인 키워드 '성형정보', '시술후기', '미녀는 괴로워' 타겟팅. Organization 스키마 적용.
- **기술 요구사항**: LCP 최적화를 위한 이미지 우선순위 로딩.
- **모바일 퍼스트**: 하단 고정 탭 바(Home, Search, Review, My) 배치.
- **법적 준수사항**: 의료법상 과도한 유인 행위 금지 문구 및 경고 문구 노출.
- **KPI**: 이탈률(Bounce Rate) 40% 이하 유지, 페이지당 체류 시간 증가.

### [Section 2] 콘텐츠 허브 (Articles Hub)
- **섹션 목적 & 사용자 여정**: 383개의 방대한 정보를 체계적으로 탐색. 사용자가 고민하는 주제(비용, 안전, 회복)별로 정보를 필터링하여 습득.
- **핵심 기능 리스트**:
  - 4대 카테고리 필터 (PRICE, SAFETY, RECOVERY, CLINIC_CHOICE)
  - 검색 기능 (자동 완성 포함)
  - 무한 스크롤 또는 페이지네이션
- **콘텐츠 전략**: 주 3회 신규 아티클 업데이트. 전문가 감수 콘텐츠 강조.
- **SEO 전략**: 카테고리별 롱테일 키워드 전략. Breadcrumbs 스키마 적용.
- **기술 요구사항**: ISR을 통한 빠른 로딩 및 최신성 유지.
- **모바일 퍼스트**: 한 손 조작이 가능한 필터 UI(Bottom Sheet).
- **법적 준수사항**: '의료 정보 제공 목적' 명시, 특정 병원 홍보 편향성 배제.
- **KPI**: 카테고리별 클릭률(CTR), 아티클 공유 횟수.

### [Section 3] 기사 상세 (Article Detail)
- **섹션 목적 & 사용자 여정**: 특정 주제에 대한 심도 있는 정보 제공 및 관련 병원/상담으로의 전환 유도.
- **핵심 기능 리스트**:
  - 목차(Table of Contents) 바로가기
  - 관련 아티클 추천 알고리즘
  - 소셜 공유 및 북마크 기능
  - 본문 내 관련 시술/병원 카드 삽입
- **콘텐츠 전략**: 텍스트 위주가 아닌 인포그래픽 및 비교 표 적극 활용.
- **SEO 전략**: Article 및 FAQ 스키마 적용. H1~H3 태그 구조화.
- **기술 요구사항**: 코드 스플리팅을 통한 본문 렌더링 가속화.
- **모바일 퍼스트**: 가독성을 고려한 폰트 크기(16px 이상) 및 줄 간격 설정.
- **법적 준수사항**: 부작용 고지 문구 필수 포함, 의료인 자문 시 성명/약력 표기.
- **KPI**: 스크롤 깊이(Scroll Depth) 70% 이상, 상담 신청 전환율.

### [Section 4] 리얼 후기 (Reviews)
- **섹션 목적 & 사용자 여정**: 실제 사용자들의 경험담을 통해 신뢰도 확보. 시술 전후 사진과 솔직한 평점을 통해 의사결정 지원.
- **핵심 기능 리스트**:
  - 시술별/병원별 후기 필터링
  - 전후 사진 비교 슬라이더
  - 후기 작성 및 평점 시스템
  - '도움이 되었어요' 추천 기능
- **콘텐츠 전략**: 사용자 생성 콘텐츠(UGC) 중심. 허위 후기 필터링 시스템 운영.
- **SEO 전략**: Review 스키마 적용으로 검색 결과 별점 노출 유도.
- **기술 요구사항**: 이미지 최적화 및 지연 로딩(Lazy Loading).
- **모바일 퍼스트**: 사진 업로드 최적화 UI.
- **법적 준수사항**: 의료법 제56조 준수 (로그인 후 후기 열람 권장 등 검토).
- **KPI**: 월간 신규 후기 수, 후기 조회수 대비 상담 전환율.

### [Section 5] 병원 찾기 (Hospitals / Clinic Directory)
- **섹션 목적 & 사용자 여정**: 위치 기반 또는 시술별 최적의 병원 탐색. 병원의 전문성, 위치, 시설 정보를 확인.
- **핵심 기능 리스트**:
  - 지도 기반 병원 검색 (Kakao/Google Map API)
  - 진료 시간 및 전문의 정보 표시
  - 병원별 보유 장비 및 특화 시술 정보
  - 실시간 상담 가능 여부 표시
- **콘텐츠 전략**: 병원별 공식 데이터 정기 업데이트 (분기별).
- **SEO 전략**: Local Business 스키마 적용. '지역명 + 시술명' 키워드 최적화.
- **기술 요구사항**: 위치 정보 API 연동 및 클러스터링 기술 적용.
- **모바일 퍼스트**: '내 주변 병원 찾기' GPS 연동 기능.
- **법적 준수사항**: 실제 병원명 사용 시 의료광고 심의 준수 (목업 단계는 가명 사용).
- **KPI**: 병원 상세 페이지 조회수, 전화 걸기/지도 보기 클릭수.

### [Section 6] 상담 신청 (Consultation / Lead Gen)
- **섹션 목적 & 사용자 여정**: 정보 탐색을 마친 사용자를 실제 고객으로 전환. 간편하고 보안이 유지되는 상담 채널 제공.
- **핵심 기능 리스트**:
  - 단계별 상담 폼 (시술 부위, 예산, 희망 날짜 등)
  - 카카오톡 상담 연동 버튼
  - 개인정보 수집 및 이용 동의 체크박스
  - 상담 신청 내역 확인 (마이페이지 연동)
- **콘텐츠 전략**: 상담 신청 시 혜택(가이드북 증정 등) 강조.
- **SEO 전략**: 전환 페이지로 설정하여 검색 노출보다는 내부 링크 유입에 집중.

- **모바일 퍼스트**: 입력이 쉬운 대형 버튼 및 입력창 UI.
- **법적 준수사항**: 개인정보보호법 준수, 민감정보 수집 시 별도 동의 절차.
- **KPI**: 상담 신청 완료율(Conversion Rate), 리드당 비용(CPL).

## 5. 출시 로드맵 (Rollout Roadmap)

### Phase 1: 기반 구축 및 핵심 마이그레이션 (1-4주)
- **주요 작업**: Next.js 프로젝트 세팅, 기존 383개 콘텐츠 마이그레이션 완료, 홈/콘텐츠 허브/기사 상세 페이지 오픈.
- **의존성**: 도메인 이전 및 리다이렉트 설정 완료 필요.

### Phase 2: 커뮤니티 및 전환 기능 강화 (5-8주)
- **주요 작업**: 리얼 후기 섹션 오픈, 상담 신청 폼 고도화, 병원 찾기 지도 서비스 연동.
- **의존성**: 병원 데이터베이스 구축 및 지도 API 키 확보.

### Phase 3: 신규 섹션 확장 (9-12주)
- **주요 작업**: 시술별 가이드, 비디오 허브, 이벤트 페이지 오픈.
- **의존성**: 영상 콘텐츠 제작 및 병원 제휴 프로모션 확정.

### Phase 4: 최적화 및 마케팅 가속화 (13주~)
- **주요 작업**: SEO 성과 분석 및 키워드 보정, Core Web Vitals 최적화, 유료 광고 캠페인 시작.
- **의존성**: Phase 1~3 데이터 분석 결과 반영.
